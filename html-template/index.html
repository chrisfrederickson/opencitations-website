Open Citation Indexes

According to Wikipedia, a citation index is a kind of bibliographic index, an index of citations between publications, allowing the user to easily establish which later documents cite which earlier documents. Several of these indexes have been published in the past. Some are freely accessible but not downloadable (e.g. Google Scholar), while others can be accessed only by paying significant access fees (e.g. Web of Science and Scopus).

Caring about setting bibliographic metadata and citation links on open, we are currently working on expand the set of services and datasets we can provide to the public. In particular, we are building several Open Citation Indexes by using the data recorded in particular bibliographic database, such as Crossref. The idea behind these indexes is to provide several datasets containing citations as first-class data entities, where each citation is identified by a particular Open Citation Identifier, or OCI - which has a simple structure: the lower-case letters "oci" followed by a colon, followed by two numbers separated by a dash (e.g. oci:1-18).

The indexes we have published are listed as follows:

  • the Crossref Open Citation Index (COCI): a new RDF dataset of more than 280 million DOI-to-DOI citations extracted from Crossref.

All the RDF data in each of the aforementioned indexes are organising according to the data model highlighted in Figure 1. This model is entirely based on the Citation Typing Ontology (CiTO) for describing the data, and the Provenance Ontology (PROV-O) for the provenance information. In particular, each citation (i.e. an individual of the class cito:Citation) is identified by the dash-separated numbers of the OCI defining it, and is further characterised by:

  • the DOI URL of the citing entity (cito:hasCitingEntity);

  • the DOI URL of the cited entity (cito:hasCitedEntity);

  • the creation date of the citation according to the ISO date format YYYY-MM-DD, which corresponds to the publication date of the citing entity (cito:hasCitationCreationDate);

  • the interval between the publication date of the cited entity and the publication date of the citing entity, expressed using the XSD duration format PnYnMnD (cito:hasCitationTimeSpan.

All the data in each Open Citation Index:

  • are available as dumps on Figshare in CSV and N-Triples formats;

  • can be directly accessed by using the HTTP URI of the citations in different formats (HTML, RDF/XML, Turtle, and JSON-LD) via content negotiation;

  • can be queried by means of a SPARQL endpoint;

  • can be retrieved by using a REST API in JSON and CSV formats, implemented by means of RAMOSE;

  • can be searched and browsed by means of the graphical facilities provided by OpenCitations, i.e. OSCAR and LUCINDA.
